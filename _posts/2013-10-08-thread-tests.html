--- layout: article title: Thread Tests date: '2013-10-08T15:12:00.000-07:00' author: aml tags: - apprenticeship modified_time: '2013-10-08T15:12:15.102-07:00' blogger_id: tag:blogger.com,1999:blog-1730093662423317857.post-6325988175413444905 blogger_orig_url: https://immutablearlandis.blogspot.com/2013/10/thread-tests.html --- I've come to a point in my code where it's time to implement multi-threading. I knew that this would be a big testing hurdle and I wasn't sure how I would handle it, but in true Agile fashion I delayed that train of thought until it was time to deal with it. The first part of the challenge is to figuring out how to test if a thread is sleeping, since implementing some sleep functionality is on my storyboard this week.

Testing Options

So one option that would work fairly well is to write a Thread interface so that I can implement a mock Thread and make some kind of boolean assertion about whether or not its "sleep" method got called. The problem with this approach is that I don't like the idea of testing against some odd state change. I would much rather prefer some kind of behavioral test, but unfortunately that doesn't seem like it's an option in this case.

I was hoping to scour the web and determine what other developers have done in similar situations but my searches thus far have yielded few promising results.
