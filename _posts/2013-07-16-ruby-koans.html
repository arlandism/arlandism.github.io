To start things off, I've downloaded the Ruby Koans. The exercises are pretty fun and don't feel like work at all.

Already, I've come across a few things that are different from my native Python. For instance, what does this do?


array = [:apples, :oranges, :bananas, :pineapple]
array[2..-1]

Apparently, it outputs: 
[:bananas, :pineapple]
That kinda makes sense, "Okay, take the 2nd index element and then the last element since the index goes to -1". But then, why isn't this equal? 
array[2..0] != [:bananas, :apples]
And what about this: 
array[4]
=> nil
array[4..100]
=> []
So, if I try to access an index that doesn't exist then I get a nil object, but if I use a Range then I get an empty array? I suspect there's some magic going on with the Range object and the Array object, and that would explain why I don't understand the first example.
(Billy Mays voice) But wait, there's more! WTF is going on here?! 
array[4,0]
=> []
array[5,0]
=> nil
It looks like if you try to access one index past the end of the array then you'll get an empty array back, but if you go past that then you get a nil object. But that only happens when you pass in a second argument to the accessing method, otherwise you just get a nil object. Weird!
